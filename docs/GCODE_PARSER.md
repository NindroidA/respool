# Respool — G-Code Parser Specification

## Overview

The spool swap calculator includes a client-side G-code parser that extracts per-layer filament extrusion data. This runs entirely in the browser via `FileReader` — no G-code data is ever sent to the server.

Supports **auto-detection** of 8 major slicers with graceful fallback for unknown formats.

---

## Supported Slicers

| Slicer | Detection Method | Layer Marker | Weight Source |
|--------|-----------------|--------------|---------------|
| **AnkerMake / EufyMake Studio** | `;Filament weight:` header | `;LAYER:N` | `;Filament weight: Xg` (grams) |
| **Cura / Creality Print** | `;Generated with Cura` | `;LAYER:N` | `;Filament used: X.Xm` (meters → converted) |
| **PrusaSlicer** | `; generated by PrusaSlicer` | `;LAYER_CHANGE` | `; filament used [g] = X` |
| **OrcaSlicer** | `; generated by OrcaSlicer` | `;LAYER_CHANGE` | `; total filament weight [g] = X` |
| **Bambu Studio** | `; BambuStudio` | `; CHANGE_LAYER` | `; total filament weight [g] = X` |
| **Simplify3D** | `; Generated with Simplify3D` | `; layer N, Z = X` | `;   Plastic weight: Xg` |
| **SuperSlicer** | `; generated by SuperSlicer` | `;LAYER_CHANGE` | `; filament used [g] = X` |
| **IdeaMaker** | `;Sliced by ideaMaker` | `;LAYER:N` | `;Material#1 Used: Xmm` (mm → converted) |

### Auto-Detection

The parser scans the first **150 lines** for slicer-specific header signatures. Detection confidence levels:
- **High**: Exact slicer header match found
- **Medium**: Layer marker style matched but no explicit header
- **Low**: No match — falls back to generic multi-pattern parser

Users can override auto-detection via a dropdown if the parser misidentifies their slicer.

### Metadata Extraction

When available in the G-code header, the parser extracts:
- **Filament density** (`; filament_density = X`) — auto-fills the density field
- **Filament material** (`; filament_type = X`) — auto-selects material
- **Filament diameter** (`; filament_diameter = X`) — uses for mm→gram conversion
- **Filament color** (`; filament_colour = #XXXXXX`) — displays color swatch
- **Tool changes** (`T0`, `T1`, etc.) — detects multi-material/AMS prints

## Critical Parsing Rules

These rules apply universally across all slicers.

### 1. Layer Detection — Profile-Specific Pattern

Each slicer uses a different comment format to mark layer transitions. The parser uses the detected slicer's `layerMarker` regex pattern.

**Do NOT count** as layer boundaries:
- Z-only G0/G1 moves (Z-hops)
- G2/G3 arc moves with Z (spiral moves)
- Informational comments (`;Z:`, `;BEFORE_LAYER_CHANGE`, etc.)

### 2. Extrusion Mode Detection — Word Boundary Match

```javascript
if (/^M83\b/.test(line)) isRelative = true;
if (/^M82\b/.test(line)) isRelative = false;
```

**Why:** Slicers append trailing comments (e.g., `M83 ; use relative distances`). Word boundary ensures the match works regardless of trailing content.

### 3. Filter Retraction — Only Count E on XY Moves

```javascript
if (/^G[01]\s/.test(line) && /[XY]/.test(line)) {
  // Only count E values here
}
```

**Why:** Retraction moves (`G1 E-3 F3600`) and unretractions (`G1 E3 F3600`) have no X/Y coordinates. Counting them inflates the total by ~60%.

### 4. Handle G92 E0 Resets

```javascript
if (/^G92\s/.test(line) && /E0/.test(line)) lastE = 0;
```

**Why:** Slicers periodically reset extruder position to prevent float precision issues.

### 5. Header Weight as Authoritative

When the slicer reports total filament weight in a header comment, use it as the authoritative total and scale per-layer values proportionally. This eliminates rounding drift across 200k+ extrusion moves.

**Fallback:** If no header weight is found, use the parsed total directly.

## Unit Conversion

Different slicers report filament usage in different units:

| Unit | Conversion | When Used |
|------|-----------|-----------|
| **grams** | Direct use | PrusaSlicer, OrcaSlicer, BambuStudio, AnkerMake |
| **mm** (length) | `length × π × (d/2)² × density / 1000` | IdeaMaker, some PrusaSlicer |
| **meters** (length) | `meters × 1000 × π × (d/2)² × density / 1000` | Cura |
| **cm³** (volume) | `volume × density` | Some Simplify3D |

When filament is reported in length units, the user **must** have the correct density set (either auto-detected from the G-code or manually selected). The parser warns if density is required but not detected.

## Error Handling

| Condition | Response |
|-----------|----------|
| Empty file | Error: "File is empty." |
| Binary file (.3mf, .stl) | Error: "This is a .3mf file, not G-code." |
| No G-code commands | Error: "No G-code commands found." |
| No layer markers | Error: "No layer markers found." + suggest manual slicer selection |
| No extrusion detected | Error: "Layers detected but no extrusion found." |
| Header/parsed mismatch >5% | Warning shown but parsing continues |
| Very large file (>50MB) | Warning: "Large file — parsing may take a few seconds" |
| Multi-material (AMS) detected | Info: "Multi-material detected. AMS handles swaps automatically." |
| Slicer not identified | Warning + manual dropdown offered |

---

## Multi-Material / AMS Detection

When the parser detects multiple tool change commands (`T0`, `T1`, `T2`, `T3`), it identifies the print as multi-material:

- **BambuStudio/OrcaSlicer with AMS**: Shows info banner explaining that filament swaps are automatic
- **Other slicers**: Still allows the calculator to run for manual multi-spool planning

---

## Per-Slicer Pause Instructions

The results display adapts its instructions based on the detected slicer:

- **EufyMake**: Layer slider + "+" button in preview
- **PrusaSlicer/OrcaSlicer/SuperSlicer**: Right-click layer bar → Add custom G-code → `M600`
- **Bambu Studio**: Right-click layer bar → Insert pause (`M601`)
- **Cura**: Extensions → Post Processing → Filament Change script
- **Simplify3D**: Edit Process → Scripts tab → Layer Change Script
- **IdeaMaker**: Edit → Modify G-code for specific layer
- **Generic**: Manual M600/M0 insertion in text editor
